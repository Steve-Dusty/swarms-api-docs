**Swarm Type**: `DebateWithJudge`

## Overview

The DebateWithJudge swarm type implements a structured debate format where multiple agents present and defend opposing viewpoints on a topic, while a designated judge agent evaluates the arguments and determines the most compelling position. This architecture is ideal for exploring complex topics, stress-testing ideas, and arriving at well-reasoned conclusions through adversarial analysis.

Key features:
- **Structured Argumentation**: Agents present formal arguments for their positions
- **Adversarial Analysis**: Opposing viewpoints challenge assumptions
- **Impartial Evaluation**: Judge agent weighs evidence objectively
- **Transparent Reasoning**: Clear rationale for final decisions

## Use Cases

- Policy analysis and decision-making
- Technical architecture reviews
- Investment thesis validation
- Risk assessment with multiple perspectives
- Academic and research debates
- Legal argument evaluation

## Best Practices

- Assign clear, opposing positions to debater agents
- Use a lower temperature (0.2-0.4) for the judge to ensure consistent evaluation
- Set `max_loops` to 2 or more to allow for rebuttals
- Include specific evaluation criteria in the judge's system prompt
- Design debaters with comparable expertise levels for balanced arguments
- Use the `role` parameter to distinguish debaters from the judge
