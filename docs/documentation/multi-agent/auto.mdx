**Swarm Type**: `auto`

## Overview

The `auto` swarm type provides intelligent, automatic selection of the optimal swarm architecture based on your task description and requirements. Instead of manually choosing between SequentialWorkflow, ConcurrentWorkflow, MixtureOfAgents, or other architectures, the system analyzes your task and selects the most appropriate swarm type automatically.

Key features:
- **Intelligent Selection**: Analyzes task to determine optimal architecture
- **Zero Configuration**: Just describe your task and let the system decide
- **Adaptive**: Considers task complexity, agent requirements, and workflow patterns
- **Beginner-Friendly**: Perfect for users new to multi-agent systems

## Use Cases

- Users unfamiliar with swarm architecture selection
- Rapid prototyping and experimentation
- Tasks with unclear workflow requirements
- General-purpose multi-agent applications
- Exploratory analysis and research

## How Auto Selection Works

The system analyzes your task based on several factors:

| Factor | Consideration |
|--------|---------------|
| **Task Dependencies** | Sequential tasks → SequentialWorkflow |
| **Parallel Potential** | Independent subtasks → ConcurrentWorkflow |
| **Collaboration Needs** | Discussion required → GroupChat |
| **Decision Making** | Voting/consensus needed → MajorityVoting |
| **Multi-perspective** | Diverse expertise → MixtureOfAgents |
| **Task Routing** | Different task types → MultiAgentRouter |

## Best Practices

- Provide detailed task descriptions for better architecture selection
- Review the `selected_swarm_type` in responses to learn which architectures work best
- Once you identify optimal patterns, consider using specific swarm types directly
- Use `auto` for experimentation and prototyping
- Check `selection_reason` to understand the system's decision
