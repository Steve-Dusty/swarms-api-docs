**Swarm Type**: `RoundRobin`

## Overview

The RoundRobin swarm type distributes tasks across agents in a rotating, circular pattern. Each agent receives tasks in turn, ensuring even workload distribution and preventing any single agent from becoming a bottleneck. This architecture is ideal for processing multiple similar tasks where load balancing is important.

Key features:
- **Even Distribution**: Tasks are distributed equally among agents
- **Load Balancing**: Prevents overloading individual agents
- **Predictable Assignment**: Clear, deterministic task routing
- **Scalable**: Easily add more agents to increase throughput

## Use Cases

- Batch processing of similar tasks
- Customer support ticket routing
- Document processing pipelines
- API request load balancing
- Parallel content generation
- Data validation across multiple validators

## Best Practices

- Use RoundRobin for homogeneous tasks that can be handled by any agent
- Configure agents with identical capabilities for consistent results
- Scale the number of agents based on your task volume
- Consider task complexity when determining agent count
- Use `tasks` array (plural) for multiple tasks to be distributed
- Monitor distribution metrics to ensure even load balancing
